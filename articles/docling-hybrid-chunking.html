Docling Hybrid Chunking

最終更新: 2026-03-24 17:53 JST
Docling Hybrid Chunking は、文書を検索しやすい「かたまり（チャンク）」に分ける仕組みです。ただ同じ長さで切るのではなく、見出し・要約・画像のテキストなどを活かして、検索に役立つ情報がついたチャンクを作ります。

Docling Hybrid Chunking は、ドキュメント Ingest Pipeline の中で、文書をチャンク（小さなかたまり）に分ける役割を持っています。コードでは job_src.chunking が中心です。Project Diana Agentic RAG の中で、もとの文書を Policy-Enforced Retrieval（ルールに従って検索する仕組み）で使いやすい形に変えます。

なぜ Hybrid Chunking なのか

この仕組みのねらいは、文書をただ短く切ることではありません。検索用の短い要約、回答を作るための広めの文脈、見出しの並び、キーワード、画像から読み取ったテキストなどをまとめて、ひとつの検索単位にすることです。チャンク分割は地味な作業に見えますが、検索の質を大きく左右します。

同じトークン数でずらしながら切る方法（sliding window）は簡単ですが、見出しの区切りや表のまとまりを壊しやすいです。Project Diana では Docling の HybridChunker を使い、文書の構造に合わせて分割します。大きすぎるチャンクは後から追加で分けます。これは Split Deployment RAG の考え方で、手間をかけてでも検索素材の質を上げるやり方です。

Docling Hybrid Chunking の流れ: まず見出しがなければ補い、HybridChunker で分割します。次に画像テキスト(OCR)を割り当て、大きすぎるチャンクをさらに分けて、情報つきチャンクが完成します。

構造情報をどう保持するか

まず、見出しがない文書には自動で仮の見出しをつけます。こうしておくと、見出しをもとにした文脈の取り出しがいつでもうまく動きます。次に HybridChunker が文書の構造に沿ってチャンクを作り、見出しの階層を search_headings として保存します。

こうする理由は、回答を作るときに「この文章はどの節の中にあるか」という情報をなくさないためです。Context Full Construction で詳しく説明していますが、チャンクの短い本文だけでなく、まわりの節構造もたどれるようにしておくことで、検索結果の質が安定します。

検索用の enrichment

チャンク処理では、もとのテキストをそのまま返すだけでは終わりません。OpenAI を使って要約を作り、search_text として保存します。同時にキーワードも取り出します。つまり、チャンクはただの「文書の切れはし」ではなく、「検索に必要な説明がついた切れはし」になります。

context_full は、Markdown に変換した文書から見出しの境界まで前後を広げて取り出したものです。長すぎる場合は文の区切りで短くします。全文をそのまま渡すのでもなく、短い断片だけに頼るのでもなく、回答を作るのにちょうどよい長さの文脈を用意する工夫です。詳しくは Search Summary Enrichment や Context Full Construction をご覧ください。

Picture OCR との結合

チャンク処理では、Picture OCR Stage（画像からテキストを読み取る処理）の結果を、関係するチャンクにひもづけます。ここで大事なのは、OCR テキストを文書全体にばらまくのではなく、画像の ID やページ番号を手がかりにして「この画像はどのチャンクのものか」を判定していることです。こうすると、関係のないチャンクに画像のテキストが混ざりにくくなります。

大きすぎるチャンクを追加で分けた場合も、OCR テキストは最初の分割チャンクにだけ入れます。こうすることで、同じ OCR テキストが複数のチャンクに重複して入って検索順位がおかしくなるのを防いでいます。

Oversize split と fallback

構造に合わせて分割するだけでは、とても大きな表や長い節には対応しきれません。そこで、トークン数の上限を超えたチャンクを見つけたら、追加で分割します。表の場合はヘッダー行を残しつつ行単位で分け、ふつうのテキストなら文の区切りで分けます。詳しくは Chunk Oversize Splitting をご覧ください。

また、チャンク処理の前の段階には Parse Mode Fallback があります。これは高速パースと OCR パースを切り替える仕組みです。つまり、チャンクの出来はチャンク処理だけでなく、パース方法や OCR の品質にも左右されます。

Retrieval への含意

ここで作られたチャンクは、検索のときに回答の根拠候補として使われます。また、回答の中で引用としても再利用されます。そのため、チャンクの質は検索の網羅性だけでなく、引用の自然さや回答の正確さにも直接つながります。ここで作った情報つきチャンクが Policy-Enforced Retrieval の入力になるので、チャンク分割と検索は別の工程ですが、とても密接に関係しています。
